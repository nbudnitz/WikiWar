    Methodology

    Live data comes from Wikimedia EventStreams. API calls are used only for live enrichment, catch-up, and drill-downs. Historical scoreboards should be produced from downloadable Wikimedia datasets, not multi-year API crawling.

    Historical scores are evidence-gated. A page needs repeated article-text battles with at least two combatants before it can rank highly; raw revert volume is supporting context, not proof by itself.

    Historical scoreboards are grouped by year. Monthly dump partitions remain the durable backfill unit, and each completed month is folded into its year from local aggregate tables when the scoreboard is queried.

    Historical rankings can be switched between Method 1: Edit War Priority, which preserves the strict mutual-revert scoreboard; Method 2: Page War Priority, which also admits pages with heavy raw revert activity, broad multi-editor churn, and talk-page activity; and Method 3: Most Discussed, which ranks by observed talk-page byte length in the selected year; and Method 4: Governance Priority, which boosts local dump evidence of RfCs, arbitration, edit restrictions, page protection, and current protection levels.

    Battle drill-downs use targeted MediaWiki revision-content diffs. For large historical pages, WikiWar first fetches revision metadata for the selected period, then downloads content for revert-centered revision pairs so the page stays responsive. Talk-page debate matching the contested terms receives a strong boost, while vandalism or graffiti cleanup without battles is penalized.
